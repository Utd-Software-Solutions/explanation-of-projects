الشبكات

جدران حماية، عناوين IP، موازنة أحمال، وشبكات خاصة

🔌 نظرة عامة

خدمات الشبكات في UTD Cloud بتديك تحكم كامل في كيف مواردك السحابية بتتواصل مع العالم الخارجي ومع بعضها. جدران حماية لحماية سيرفراتك، عناوين IP ثابتة لخدماتك، وموازنة أحمال لتوزيع الضغط.

🛡 جدار الحماية (Firewall)

تحكم في مين يقدر يوصل لسيرفراتك ومواردك:

📥

قواعد واردة (Inbound)

  • حدد مين يقدر يوصل لسيرفرك من الخارج
  • اسمح بمنافذ معينة فقط (مثل 80, 443)
  • حجب عناوين IP مشبوهة
  • السماح لعناوين IP محددة فقط
📤

قواعد صادرة (Outbound)

  • تحكم في اتصالات السيرفر بالخارج
  • اسمح فقط بالاتصالات الضرورية
  • حماية إضافية ضد البرامج الضارة
  • مراقبة حركة البيانات الخارجة

🌏 عناوين IP

📍

IP ثابت (Static)

  • عنوان IP ما يتغيرش أبداً
  • ضروري لو عندك نطاق مربوط بالسيرفر
  • مطلوب للخدمات اللي تحتاج عنوان ثابت
  • تكلفة إضافية بسيطة يومياً
🔄

IP مؤقت (Ephemeral)

  • يتغير كل ما السيرفر يُعاد تشغيله
  • مجاني — يُعطى تلقائياً لكل سيرفر
  • مناسب للتجارب والسيرفرات المؤقتة
  • مش مناسب لو عندك نطاق مربوط

⚖ موازنة الأحمال (Load Balancer)

لما عندك أكثر من سيرفر — موازنة الأحمال بتوزّع الطلبات بينهم بالتساوي عشان مفيش سيرفر يتحمّل فوق طاقته.

❓

إيه هو؟

نظام بيقعد قدام سيرفراتك ويوزّع زيارات الموقع عليهم بالتساوي — لو سيرفر وقع، يحوّل تلقائياً للباقيين.

💡

ليه تحتاجه؟

عندك موقع زياراته كثيرة — سيرفر واحد مش كافي. موازنة الأحمال بتوزّع الضغط وتضمن إن الموقع يفضل شغال.

🛠

كيف تنشئه؟

من لوحة التحكم — اختر السيرفرات اللي تحب توزّع بينها وحدد نوع الموازنة. النظام يعمل الباقي.

🔒 الشبكات الخاصة (VPC)

🔒 الشبكة الخاصة الافتراضية (VPC): كل حسابك عنده شبكة خاصة معزولة — سيرفراتك بتتواصل مع بعض داخلياً بأمان كامل. الاتصال الداخلي بين مواردك مجاني وسريع. تقدر تنشئ شبكات فرعية إضافية لعزل البيئات (إنتاج / تجارب / تطوير).
