👥 إدارة الفريق

العميل يقدر يضيف أعضاء فريقه بأدوار مختلفة. لا حاجة لمشاركة بيانات الحساب مع أي شخص — كل عضو يدخل بحسابه الخاص وبصلاحياته المحددة.

🔒 الأدوار والصلاحيات

الدورالصلاحيات الكاملةالقيود
المالك (Owner) صلاحيات كاملة — كل شيء بدون استثناء —
مدير (Admin) إدارة المشاريع + المحفظة + الفريق + الفوترة + الخدمات لا يمكنه حذف الحساب أو تغيير المالك
مطور (Developer) إدارة المشاريع + عرض بيانات الاعتماد (API Keys) + التوثيق قراءة فقط للمعلومات المالية — لا وصول للمحفظة أو الفوترة أو الفريق
محاسب (Billing) المحفظة + الفواتير + الباقات + سجل المعاملات لا وصول للمشاريع أو المفاتيح أو الإعدادات التقنية
مشاهد (Viewer) عرض كل شيء — قراءة فقط لا يمكنه تعديل أي شيء

➕ كيف يضيف العميل عضو جديد

1

إدخال إيميل العضو

العميل يدخل البريد الإلكتروني للعضو الجديد.

2

اختيار الدور

يختار الدور المناسب: مدير، مطور، محاسب، أو مشاهد.

3

تحديد الخدمات المتاحة

يحدد أي خدمات يقدر العضو يشوفها — أو يختار "جميع الخدمات".

4

إرسال الدعوة

يضغط "إرسال الدعوة" ← العضو يستلم إيميل بالرابط.

5

العضو ينضم

العضو ينشئ حسابه ← يدخل الداشبورد بصلاحياته المحددة فوراً.

🔧 تقييد على مستوى الخدمة

صلاحيات دقيقة لكل خدمة

المالك يقدر يحدد لكل عضو أي خدمات يشوفها بالضبط. مثلاً:

  • مطور مقيّد بـ Stream فقط — لا يشوف Cloud ولا OTP
  • محاسب يشوف فواتير كل الخدمات لكن ما يقدر يدخل المشاريع
  • مشاهد مقيّد بـ Stream — يشوف الإحصائيات بس لخدمة واحدة
الفائدة: التحكم الدقيق يحمي البيانات الحساسة. الفريق التقني يشوف بس اللي يخصه، وفريق المحاسبة يشوف بس الأرقام.
