⚡ خدمات UTD Stream — 5 خدمات أساسية

هذه هي الخدمات اللي بنقدمها. كل خدمة لها صفحة/قسم خاص بيها على الموقع:

📞

1. المكالمات الصوتية والفيديو

مكالمات بين شخصين أو مجموعة — صوت فقط أو صوت + فيديو بكل الجودات من 240p لحد 4K.

  • مكالمات 1-on-1 (شخص لشخص)
  • مكالمات جماعية (Group Calls)
  • صوت فقط أو صوت + فيديو
  • كل الجودات: 240p, 360p, 480p, 720p, 1080p, 4K
  • تسجيل المكالمات
  • إلغاء ضوضاء تلقائي
  • مشاركة الشاشة
📡

2. البث المباشر (Live Streaming)

بث مباشر لأعداد كبيرة من المشاهدين — مقدم البث (Host) يبث والجمهور (Audience) يشاهد ويتفاعل.

  • بث بكل الجودات حتى 4K
  • عدد مشاهدين غير محدود
  • تفاعل فوري (تعليقات، إعجابات)
  • Co-host (أكثر من مقدم)
  • تسجيل البث تلقائياً
  • مناسب للتجارة المباشرة والترفيه
🎧

3. الغرف الصوتية (Voice Rooms)

غرف صوتية تفاعلية مثل Clubhouse — مضيف يتحكم بالغرفة ومتحدثين ومستمعين.

  • غرف بأعداد كبيرة من المشاركين
  • نظام أدوار: مضيف / متحدث / مستمع
  • رفع وخفض صوت المشاركين
  • تأثيرات صوتية
  • ألعاب تفاعلية داخل الغرفة
  • شائع جداً في تطبيقات MENA
💬

4. الدردشة (Chat / Messaging)

نظام دردشة متكامل — رسائل بين شخصين وداخل مجموعات. نص، صور، ملفات، ورسائل صوتية.

  • دردشة 1-on-1 (شخص لشخص)
  • دردشة جماعية (Group Chat)
  • رسائل نصية، صور، ملفات، صوتية
  • مؤشرات الكتابة والقراءة
  • تاريخ الرسائل والبحث
  • تشفير من طرف لطرف
🔌

5. الإشارات (Signaling)

البنية التحتية لإدارة الاتصالات — إرسال/استقبال دعوات المكالمات وإدارة الحالة. مطلوبة لتشغيل المكالمات والبث.

  • إدارة دعوات المكالمات الواردة/الصادرة
  • حالة الاتصال الفورية (Online/Offline)
  • إشعارات Push
  • تزامن الحالة بين الأجهزة

📊 ثانياً: لوحة التحكم (Dashboard)

بعد ما العميل يسجل حساب، يدخل على لوحة تحكم يدير منها مشاريعه وخدماته. المطلوب بناء الأقسام التالية:

1. إنشاء مشروع جديد

السيناريو: العميل يسجل حساب ← يدخل الداشبورد ← يضغط "إنشاء مشروع جديد" ← يكتب اسم المشروع ← النظام يولّد له بيانات المشروع تلقائياً.

بيانات المشروع اللي تتولّد تلقائياً

لما العميل ينشئ مشروع، النظام لازم يولّد له البيانات التالية ويعرضها في صفحة إعدادات المشروع:

البيان الوصف ملاحظات
App ID معرّف فريد للمشروع — يستخدمه المطور في الـ SDK رقم فريد، يتولّد تلقائياً، غير قابل للتغيير
App Sign مفتاح توقيع للمصادقة من جهة العميل (Client-side) قابل لإعادة التوليد، يُخفى بعد أول عرض
Server Secret مفتاح سري للمصادقة من جهة السيرفر (Server-side) قابل لإعادة التوليد، لا يُعرض إلا مرة واحدة
Callback URL رابط يحدده العميل لاستقبال الإشعارات من النظام (أحداث المكالمات، الرسائل، إلخ) يدخله العميل يدوياً، اختياري
Callback Secret مفتاح لتوثيق الـ Callbacks (التأكد إن الإشعار جاي من UTD فعلاً) يتولّد تلقائياً، قابل لإعادة التوليد

2. تفعيل الخدمات

صفحة تفعيل الخدمات داخل المشروع

كل مشروع فيه صفحة لتفعيل/تعطيل الخدمات. العميل يختار الخدمات اللي محتاجها فقط:

الخدمة الحالة الافتراضية اللي بيتفعل معاها
المكالمات الصوتية مُعطّلة SDK Voice — مكالمات صوتية 1-on-1 وجماعية
مكالمات الفيديو مُعطّلة SDK Video — فيديو بكل الجودات (240p-4K) + مشاركة شاشة
البث المباشر مُعطّلة SDK Live — بث Host/Audience + تسجيل
الغرف الصوتية مُعطّلة SDK Voice Room — غرف + أدوار + تأثيرات صوتية
الدردشة مُعطّلة SDK Chat — رسائل 1-on-1 + Group + ملفات + صور
ملاحظة: كل خدمة لها toggle (تفعيل/تعطيل). العميل يدفع فقط على الخدمات المفعّلة واللي استخدمها فعلاً.

3. لوحة المراقبة والاستخدام (Usage Dashboard)

البيانات المطلوب عرضها في الداشبورد

العميل لازم يشوف استخدامه لحظة بلحظة:

  • دقائق الصوت المستخدمة — إجمالي الشهر + رسم بياني يومي
  • دقائق الفيديو المستخدمة — مقسّمة حسب الجودة (HD / Full HD / 4K)
  • دقائق البث المباشر — مقسّمة: Host + Audience
  • عدد المستخدمين النشطين للدردشة (MAU) — الشهر الحالي
  • التكلفة التقديرية للشهر الحالي — بالدولار، محسوبة من الاستخدام الفعلي
  • تاريخ الفواتير السابقة — قائمة بالفواتير الشهرية مع إمكانية التحميل PDF

4. إعدادات المشروع (Project Settings)

صفحة الإعدادات لكل مشروع

  • معلومات المشروع: اسم المشروع (قابل للتعديل)، App ID (للقراءة فقط)، تاريخ الإنشاء
  • المفاتيح: App Sign + Server Secret — مع زر "إعادة توليد" لكل واحد (مع تحذير إن القديم يتوقف)
  • Callback Settings: حقل لإدخال Callback URL + عرض Callback Secret + زر "اختبار الاتصال"
  • إعدادات الجودة الافتراضية: اختيار جودة الفيديو الافتراضية للمشروع (720p مثلاً) — العميل يقدر يغيرها من الـ SDK برضو
  • حدود الاستخدام (Rate Limits): العميل يقدر يحط حد أقصى شهري للدقائق/التكلفة عشان ما يتفاجأش بفاتورة كبيرة
  • حذف المشروع: زر حذف مع تأكيد مزدوج

العميل ما يشتري باقة — يدفع فقط على اللي استخدمه بأسعار الـ 1,000 دقيقة العادية:

  • صوت: $0.49 لكل 1,000 دقيقة
  • فيديو HD: $1.99 لكل 1,000 دقيقة
  • فيديو FHD: $6.49 لكل 1,000 دقيقة
  • الدردشة: $49 لكل 10,000 MAU
  • ... وهكذا حسب جدول الأسعار

مناسب للعملاء اللي استخدامهم متذبذب أو مش عارفين كم هيستخدموا.

النموذج الثاني: الباقات (Packages)

العميل يشتري باقة جاهزة فيها تشكيلة من الخدمات بخصم. الباقة ممكن تشمل خدمة واحدة أو أكثر:

  • كل باقة فيها عدد محدد من دقائق الصوت + دقائق الفيديو + رسائل الدردشة (حسب الباقة)
  • الباقة بخصم عن السعر العادي (pay-as-you-go)
  • لما الباقة تخلص — الاستخدام الزيادة يتحسب بسعر الـ pay-as-you-go العادي
  • الباقات شهرية أو سنوية
للفريق: الباقات وأسعارها وتفاصيلها سيتم تحديدها لاحقاً. المطلوب بناء البنية التحتية اللي تدعم: إنشاء باقات بمكونات مرنة (X دقيقة صوت + Y دقيقة فيديو + Z MAU) + سعر الباقة + خصم عن السعر العادي.

هيكل المحافظ:
— المحفظة الأم (UTD Wallet): محفظة رئيسية على مستوى حساب العميل في UTD Software Solutions. العميل يشحنها بالطرق المتاحة (Stripe، تحويل بنكي، عملات رقمية، وغيرها).
— محفظة كل خدمة: كل خدمة لها محفظة فرعية خاصة (محفظة Stream، محفظة OTP، محفظة Cloud...). العميل ينقل الرصيد من المحفظة الأم للمحفظة الفرعية اللي يبيها.
— الخصم اليومي يتم من محفظة الخدمة، مش من المحفظة الأم مباشرة.
1

العميل يشحن المحفظة الأم (UTD Wallet)

من داشبورد UTD Software Solutions الرئيسي، العميل يضيف رصيد بالدولار عبر: Stripe (بطاقة ائتمان)، تحويل بنكي، عملات رقمية، أو أي طريقة متاحة.

2

تحويل الرصيد لمحفظة UTD Stream

العميل يدخل على داشبورد UTD Stream ← يضغط "شحن محفظة Stream" ← يحدد المبلغ ← المبلغ ينتقل من المحفظة الأم لمحفظة Stream فوراً.

2

النظام يسجل الاستخدام يومياً

كل يوم الصبح، النظام يحسب استخدام اليوم اللي فات. مثال: اليوم 12 الصبح ← يتحسب استخدام يوم 11 (200,000 دقيقة صوت + 100,000 دقيقة فيديو).

3

الخصم التلقائي من المحفظة

النظام يحسب التكلفة ويخصمها تلقائياً. لو العميل عنده باقة ← يتخصم من رصيد الباقة الأول. لو ما عندوش أو الباقة خلصت ← يتخصم من المحفظة بسعر الـ pay-as-you-go.

4

الفاتورة الشهرية (ملخص)

أول كل شهر تصدر فاتورة شهرية تفصيلية — ملخص كل الاستخدام والخصومات اليومية. الفاتورة للتوثيق (الدفع خلاص تم يومياً). قابلة للتحميل PDF.

صفحة محفظة Stream في الداشبورد — المطلوب عرضه

  • رصيد محفظة Stream: رقم واضح بالدولار في أعلى الصفحة
  • رصيد المحفظة الأم: يظهر بجانبه عشان العميل يعرف كم متاح للتحويل
  • زر "تحويل رصيد من المحفظة الأم": يحدد المبلغ ← ينتقل فوراً من الأم لـ Stream
  • سجل المعاملات اليومية: جدول بكل يوم — التاريخ | الاستخدام (صوت/فيديو/دردشة) | التكلفة | المصدر (محفظة أو باقة) | الرصيد المتبقي
  • رصيد الباقة (لو موجودة): كم دقيقة متبقية من الباقة لكل خدمة
  • تنبيه الرصيد المنخفض: إشعار (إيميل + داخل الداشبورد) لما رصيد محفظة Stream ينزل تحت حد معين (العميل يحدد الحد)
  • تنبيه الرصيد صفر: لو رصيد محفظة Stream وصل صفر والباقة خلصت ← خدمة Stream تتوقف + إشعار فوري
  • الفواتير الشهرية: فواتير Stream فقط — قائمة مع زر تحميل PDF
ملاحظة للفريق — المحفظة الأم: المحفظة الأم وطرق الشحن (Stripe، عملات رقمية، إلخ) هي مكوّن مشترك على مستوى UTD Software Solutions — مش خاص بـ Stream. لازم تتبنى مرة واحدة في الداشبورد الرئيسي وكل الخدمات تستخدمها.
