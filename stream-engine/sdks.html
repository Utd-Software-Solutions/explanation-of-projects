📦 الـ SDKs المطلوبة

المطور يحتاج SDK-ين يعملان معاً — واحد للميديا (صوت وفيديو) وواحد للاتصال الفوري (دردشة وحضور وsignaling).

UTD ZIM SDK (utd_zim)

مكتبة Flutter (Dart) رسمية — مبنية بواسطة UTD. التسمية تتبع نمط ZEGOCLOUD (zego_zim → utd_zim). تتصل بـ WebSocket Server. توفر: Presence (تتبع حضور)، Signaling (إشعارات المكالمات)، Messaging (رسائل فورية)، Conversations (إدارة المجموعات). الفئة الرئيسية: UtdZim.

LiveKit Client SDK

من مشروع LiveKit المفتوح المصدر. تتصل بـ LiveKit Media Server عبر WebRTC. مسؤولة عن نقل الصوت والفيديو بين المستخدمين. متاحة لـ: Flutter, Web, Android, iOS, React Native.

🔄 كيف يعملان معاً

utd_zim (UTD ZIM SDK)

→ WebSocket → دردشة + حضور + إشعارات مكالمات

livekit_client (LiveKit SDK)

→ WebRTC → صوت + فيديو

المطور يستخدم كلا الـ SDKs معاً — ZIM للـ signaling والدردشة، LiveKit للميديا. مثل ما ZEGOCLOUD عندهم ZIM SDK + Express SDK.

⚙ ما يقدمه UTD ZIM SDK

الخدمةما يفعله
Presenceاستعلام حالة المستخدمين (متصل/غير متصل)، الاشتراك في تحديثات الحضور لحظياً، إلغاء الاشتراك
Signalingإرسال دعوة مكالمة، استقبال الدعوة، قبول/رفض/مشغول/إنهاء — كل lifecycle المكالمة
Messagingإرسال رسائل (نص/صورة/ملف/صوت/فيديو/مخصص)، استرجاع رسالة، تحميل سجل الرسائل
Conversationsإنشاء مجموعات، إضافة/إزالة أعضاء، تحديث معلومات المجموعة، كتم، حذف، مغادرة

🚀 خطوات البدء

1

إنشاء مشروع

من لوحة تحكم UTD Stream — والحصول على App ID + App Secret.

2

تثبيت الـ SDKs

في مشروع Flutter: تثبيت utd_zim + livekit_client.

3

طلب Tokens من السيرفر

في سيرفر المطور: استخدام App ID + Secret لطلب Tokens من Engine API.

4

تهيئة الـ SDKs

في التطبيق: تهيئة UtdZim بـ Signal Token + تهيئة LiveKit بـ Media Token.

5

البدء بالبناء

البدء ببناء الميزات: غرف صوتية، بث مباشر، مكالمات، دردشة.
