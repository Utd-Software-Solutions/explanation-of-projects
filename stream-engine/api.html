واجهات API

نظرة عامة على كل الـ APIs المتاحة — ماذا يفعل كل endpoint

🔌 Client API — للمطورين

يستدعيها سيرفر المطور (Backend-to-Backend) — لأن المفتاح السري يجب أن يبقى في السيرفر. المصادقة: App ID + App Secret في Headers لكل طلب.

المجالما يمكن فعله
التوكنات (Tokens)توليد access token لدخول مستخدم لغرفة — مع تحديد الخدمة (rooms أو streaming) والدور (host, guest, audience, visitor)
الغرف (Rooms)عرض الغرف النشطة، تفاصيل غرفة مع قائمة المشاركين، إغلاق غرفة قسرياً، طرد مشارك
المكالمات (Calls)إنشاء مكالمة صوت أو فيديو، تأكيد الرنين، قبول، رفض، "مشغول"، إنهاء المكالمة، عرض حالة مكالمة، قائمة المكالمات
معلومات المشروععرض إعدادات وحالة المشروع الحالي

⚡ WebSocket API — للاتصال الفوري

يتصل بها تطبيق المستخدم عبر UTD ZIM SDK أو مباشرة. المصادقة: Signal Token عند الاتصال.

المجالما يمكن فعله
الدردشة (Messaging)إرسال رسائل، استرجاع رسائل، تحميل رسالة كمقروءة، إنشاء مجموعات، إضافة/إزالة أعضاء، تحديث مجموعة، كتم/حذف محادثة
الحضور (Presence)استعلام حالة مستخدمين، اشتراك/إلغاء اشتراك في تحديثات الحضور
المكالمات (Signaling)إرسال دعوة مكالمة، رنين، قبول، رفض، مشغول، إنهاء — كل lifecycle المكالمة

📚 REST APIs للسجل — البيانات التاريخية

الطلبالوصف

GET /conversations/{id}/messagesسجل الرسائل (مع pagination)
GET /conversations/{id}/membersقائمة أعضاء المحادثة
DELETE /messages/{id}حذف رسالة من السيرفر

🔧 Admin API — للاستخدام الداخلي

تنبيه: هذه APIs للاستخدام الداخلي من قبل فريق UTD فقط — ليست للمطورين. المصادقة: مفاتيح إدارية خاصة (ليست نفس مفاتيح المشروع).
المجالما يمكن فعله
العملاءإنشاء، عرض، تعديل، تعليق/تفعيل عميل
المشاريعإنشاء، عرض، تعديل إعدادات (الخدمات المفعلة، الجودة القصوى، عدد الغرف، timeouts)، تعليق/تفعيل، تجديد المفاتيح
لوحة المعلوماتإحصائيات عامة عن المنصة (غرف نشطة، مكالمات، مشاركين)
الاستخدامتقارير استخدام مفصلة لكل مشروع (دقائق صوت، فيديو حسب الجودة، مكالمات، رسائل)
المراقبةعرض الغرف والمكالمات النشطة حالياً، إغلاق غرفة أو مكالمة
الحظرقائمة المستخدمين المحظورين، إزالة حظر
السجلاتسجلات كل طلبات API (الـ endpoint، وقت الاستجابة، حالة الرد)
